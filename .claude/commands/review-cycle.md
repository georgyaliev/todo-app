Код уже написан. Задача: $ARGUMENTS (если не указано — возьми текущие незакоммиченные изменения, `git diff`).

Прогони через сокращённый пайплайн:

1. **Ревью кода** — делегируй `code-reviewer`. Если критичные проблемы — сообщи пользователю и спроси, чинить ли через `developer` сразу или дать рекомендации.
2. **Тестирование** — делегируй `test-engineer` для покрытия тестами и прогона полного набора.
3. **DevOps** — делегируй `devops-engineer`, только если пользователь просил подготовку к деплою/CI. Иначе пропусти этот шаг.

Дай краткое резюме по итогу: статус ревью, статус тестов, что осталось сделать.
