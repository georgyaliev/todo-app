Приложение — простой веб-to-do лист с нуля (MVP). Стек не задан.

**Продуктовые соглашения:**
- Три базовые операции: добавление задачи, отметка выполнена/не выполнена (toggle), удаление.
- Единица данных называется "задача" (не "дело", не "item").
- Статусы: "не выполнена" / "выполнена".
- Порядок отображения: по дате добавления (новые снизу).
- Лимит: до 500 задач, длина названия до 255 символов.

**Why:** Проект учебный / pet-project, одиночный пользователь, минимальный порог входа.

**How to apply:** При следующих итерациях не вводить новые сущности (проекты, теги и т.д.) без явного запроса. Редактирование задач, сортировка, фильтрация — вне рамок MVP.

**Открытый вопрос (критический):** localStorage vs серверный бэкенд — не решено. Рекомендация зафиксирована: сначала localStorage.
