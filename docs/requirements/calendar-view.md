# Calendar View — Задачи с привязкой к дате

## Проблема и цель

Текущий Todo List хранит задачи без привязки к дате, что делает его бесполезным для планирования. Пользователь не может ответить на вопрос «что мне нужно сделать в следующий вторник?».

Цель итерации — добавить временно́е измерение к задачам: каждая задача привязана к конкретному дню и отображается в виде месячной сетки, аналогичной Google Calendar в режиме «Месяц» — но без временны́х слотов. Это todo-лист с датами, не event-менеджер.

Бизнес-ценность: пользователь видит загруженность каждого дня в контексте месяца, может планировать вперёд и не упускать задачи с конкретными дедлайнами.

---

## Пользовательские истории

- Как пользователь, я хочу видеть задачи в сетке месяца, чтобы понимать, на какие дни у меня что запланировано.
- Как пользователь, я хочу добавить задачу к конкретному дню, кликнув на ячейку, чтобы не вводить дату вручную.
- Как пользователь, я хочу переключаться между месяцами, чтобы видеть задачи в прошлом и планировать на будущее.
- Как пользователь, я хочу отметить задачу выполненной прямо в ячейке дня, чтобы не переключаться на отдельный экран.
- Как пользователь, я хочу удалить задачу из ячейки дня, чтобы убрать неактуальное.
- Как пользователь, я хочу сразу видеть, что сегодня — чтобы легко ориентироваться в сетке.
- Как пользователь, я хочу видеть количество невыполненных задач в ячейке дня одним взглядом, чтобы оценить загруженность без раскрытия списка.

## Функциональные требования

### FR-0. Дизайн-система: монохромная палитра

Весь интерфейс Calendar View использует только нейтральную (чёрно-белую/серую) палитру. Единственные исключения — счётчики выполнения (FR-8).

Допустимые значения:

| Элемент | Значение |
|---|---|
| Фон ячейки (текущий месяц) | #ffffff |
| Фон ячейки (хвост) | #f5f5f5 |
| Текст задачи (невыполненная) | #1a1a1a |
| Текст задачи (выполненная) | #9e9e9e + strikethrough |
| Номер дня (обычный) | #333333 |
| Номер дня (сегодня) | #ffffff на круге #000000 |
| Номер дня (хвост) | #c0c0c0 |
| Бордеры сетки | #e0e0e0 |
| Строка «+N ещё» | #757575 |
| Кнопки навигации, чекбокс, «×» | чёрный/серый без цвета |
| Hover-состояния | фон #f0f0f0, не цветной |
| Focus-кольцо (клавиатура) | чёрный outline, не браузерный синий дефолт |

Запрещено в сетке: любые оттенки синего, зелёного, красного, оранжевого и прочих ненейтральных цветов — ни в фонах, ни в задачах, ни в кнопках.

### FR-1. Сетка месяца

- Экран отображает сетку 7 × 5 или 7 × 6 (в зависимости от месяца): 7 дней в строке (Пн–Вс), полные недели включая «хвосты» предыдущего и следующего месяца.
- Над сеткой — заголовок: «Июнь 2026» + кнопки «‹» (предыдущий месяц) и «›» (следующий месяц).
- Ячейки «хвостов» (дни вне текущего месяца) визуально приглушены — отображаются, но отличаются цветом.
- Сегодняшняя ячейка визуально выделена (обводка или закрашенный номер дня).
- При открытии приложения отображается текущий месяц.

### FR-2. Ячейка дня

- Каждая ячейка содержит: номер дня (в левом верхнем углу) и список задач этого дня.
- Задачи в ячейке отображаются в порядке добавления (новые снизу).
- Если в ячейке больше N задач (N = 3 — конкретное значение уточняется в открытых вопросах), остальные скрыты и заменены счётчиком «+K ещё».
- Выполненные задачи визуально отличаются от невыполненных (зачёркнутый текст + сниженная непрозрачность), как в текущем MVP.

### FR-3. Добавление задачи к дню

- Клик на ячейку дня открывает inline-форму или мини-попап рядом с ячейкой.
- Форма содержит: текстовое поле, кнопку подтверждения (или Enter), кнопку отмены (Escape).
- После подтверждения задача появляется в ячейке, форма закрывается.
- Правила валидации не изменяются: непустой текст, 1–255 символов.
- Добавление задачи к дням «хвостов» (вне текущего месяца) разрешено — приложение переключается на соответствующий месяц после добавления.

### FR-4. Операции с задачей в ячейке

- Toggle выполнения: клик на задачу (или чекбокс рядом с ней) переключает статус.
- Удаление: кнопка «×» у задачи (появляется при hover или всегда видна — уточняется в открытых вопросах).
- Обе операции работают без перехода на отдельный экран.

### FR-5. Навигация по месяцам

- Кнопки «‹» и «›» переключают месяц назад/вперёд.
- Кнопка «Сегодня» возвращает к текущему месяцу.
- Ограничений на глубину навигации нет (можно уйти на несколько лет вперёд или назад).

### FR-6. Модель данных — расширение

- К существующей структуре `Todo` добавляется поле `dueDate: string` (ISO-дата, `YYYY-MM-DD`).
- Поле обязательно при создании новой задачи; без даты задача не создаётся.
- Существующие задачи в localStorage без поля `dueDate` обрабатываются: при загрузке им автоматически присваивается сегодняшняя дата (миграция).
- Порядок задач внутри дня: по `createdAt` (append-only, без сортировки пользователем).

### FR-7. Персистентность

- Поведение без изменений: localStorage, ключ `"todos"`, JSON. Меняется только структура каждого объекта (добавляется `dueDate`).

### FR-8. Счётчики выполнения за отображаемый месяц

- Над сеткой (под строкой навигации) отображаются два счётчика:
  - «Выполнено: N» — задачи с `completed = true` и `dueDate` в отображаемом месяце
  - «В работе: N» — задачи с `completed = false` и `dueDate` в отображаемом месяце
- Счётчики привязаны к месяцу сетки, а не к текущему календарному месяцу.
- Задачи из «хвостов» (дни вне диапазона отображаемого месяца) в счётчики не входят.
- Счётчики обновляются немедленно при: переключении месяца, toggle completed, добавлении задачи, удалении задачи.
- При 0 задач счётчики показывают 0, не скрываются и не заменяются прочерком.
- Цвет счётчиков — единственное исключение из монохромной палитры (FR-0):
  - «Выполнено» — зелёный (#4caf50, WCAG AA на белом фоне)
  - «В работе» — синий (#2196f3, WCAG AA на белом фоне)
  - Цвет применяется к тексту счётчика целиком.

## Нефункциональные требования

| Категория | Требование |
|---|---|
| Производительность | Переключение месяца — перерисовка сетки < 100 мс. Целевой объём данных: до 500 задач суммарно по всем датам |
| Адаптивность | Сетка корректно отображается от 320px (мобильный) до 1920px (десктоп). На мобильных ячейки могут быть уменьшены, текст задач — обрезан через ellipsis |
| Доступность | Навигация по месяцам (кнопки «‹», «›», «Сегодня») доступна с клавиатуры. Форма добавления закрывается на Escape |
| Совместимость | Те же браузеры: Chrome, Firefox, Safari (последние 2 мажорные версии) |
| Миграция данных | Существующие задачи без `dueDate` не теряются при обновлении приложения |

## Ключевые UX-решения

### Сетка

Стандартная сетка в стиле Google Calendar «Месяц»: шапка с аббревиатурами дней недели (Пн, Вт, Ср, Чт, Пт, Сб, Вс), под ней ячейки одинакового размера. Высота ячейки фиксированная (ориентир: 120px на десктопе) — задачи, не влезающие по высоте, скрываются за «+K ещё».

### Добавление задачи

Клик на свободное пространство ячейки (не на задачу) → появляется текстовое поле прямо внутри ячейки или во всплывающем минипанели рядом. Дата берётся автоматически из ячейки — пользователь вводит только текст. Подтверждение — Enter или кнопка «Добавить». Отмена — Escape или клик вне формы.

Отдельной кнопки «+ Добавить задачу» с выбором даты из date picker нет — дата всегда берётся из контекста ячейки.

### Отображение задач в ячейке

Каждая задача — однострочная строка с чекбоксом слева и крестиком справа. Текст обрезается ellipsis при переполнении ширины. Выполненная задача: зачёркнутый текст + opacity снижена. Максимум 3 задачи видимы; если больше — строка «+2 ещё» в конце ячейки (кликабельность этой строки — в открытых вопросах).

### Мобильная адаптация

На экранах до 480px ячейки компактные: только номер дня и точки/счётчик задач (не полный текст). Полный список задач дня — по клику на ячейку (дополнительный экран или выдвижная панель снизу). Это упрощение допустимо для первой итерации.

## Критерии приёмки

### Сетка месяца
- [ ] Given: пользователь открывает приложение — When: страница загружена — Then: отображается сетка текущего месяца с корректным числом недель (5 или 6)
- [ ] Given: в сетке — When: любой день — Then: ячейки хвостов (вне текущего месяца) отображаются приглушённым цветом
- [ ] Given: сегодняшняя дата — When: сетка отрисована — Then: ячейка сегодняшнего дня визуально выделена среди остальных
- [ ] Given: заголовок месяца — When: сетка текущего месяца — Then: заголовок отображает название месяца и год на русском (например, «Июнь 2026»)

### Навигация
- [ ] Given: пользователь нажимает «›» — When: выполнено — Then: сетка переключается на следующий месяц, заголовок обновляется
- [ ] Given: пользователь нажимает «‹» — When: выполнено — Then: сетка переключается на предыдущий месяц
- [ ] Given: пользователь находится не в текущем месяце — When: нажимает «Сегодня» — Then: сетка возвращается к текущему месяцу

### Добавление задачи к дню
- [ ] Given: пользователь кликает на ячейку дня — When: клик — Then: появляется форма ввода задачи
- [ ] Given: форма открыта, пользователь вводит непустой текст и нажимает Enter — When: подтверждение — Then: задача появляется в ячейке этого дня, форма закрывается
- [ ] Given: форма открыта — When: пользователь нажимает Escape — Then: форма закрывается без создания задачи
- [ ] Given: форма открыта, поле пустое — When: пользователь нажимает Enter — Then: задача не создаётся
- [ ] Given: задача добавлена — When: страница перезагружена — Then: задача сохранена с правильной датой и отображается в той же ячейке

### Операции с задачей
- [ ] Given: задача в ячейке не выполнена — When: пользователь нажимает на чекбокс/задачу — Then: задача помечена выполненной (зачёркнута)
- [ ] Given: задача в ячейке выполнена — When: пользователь нажимает на чекбокс/задачу — Then: задача возвращается в статус «не выполнена»
- [ ] Given: пользователь нажимает «×» у задачи — When: выполнено — Then: задача удаляется из ячейки, остальные задачи дня не затронуты

### Лимит отображения в ячейке
- [ ] Given: в ячейке 4 и более задач — When: ячейка отрисована — Then: видимы первые 3, под ними строка «+N ещё»
- [ ] Given: в ячейке 3 и менее задач — When: ячейка отрисована — Then: все задачи видимы, строки «+N ещё» нет

### Миграция данных
- [ ] Given: в localStorage есть задачи без поля `dueDate` (старый формат) — When: пользователь открывает обновлённое приложение — Then: задачи не потеряны, им присвоена сегодняшняя дата

### Монохромный дизайн (FR-0)
- [ ] Given: сетка отрисована — When: визуальная проверка — Then: ни одна ячейка, задача, кнопка навигации или бордер не содержит ненейтрального цвета (кроме счётчиков)
- [ ] Given: задача выполнена — When: отображается в ячейке — Then: она серая и зачёркнута, без зелёного или любого цветного акцента
- [ ] Given: ячейка сегодняшнего дня — When: отображается — Then: выделена чёрным (обводка или заливка номера), не цветным
- [ ] Given: hover на кнопке или ячейке — When: пользователь наводит курсор — Then: подсветка серая (#f0f0f0), не цветная
- [ ] Given: элемент получает фокус клавиатурой — When: :focus-visible — Then: focus-кольцо чёрное, не браузерный синий

### Счётчики месяца (FR-8)
- [ ] Given: июнь 2026, 5 задач выполнено и 3 в работе — When: страница отрисована — Then: «Выполнено: 5» зелёным и «В работе: 3» синим
- [ ] Given: переключение на месяц без задач — When: месяц отрисован — Then: «Выполнено: 0» и «В работе: 0» (оба показаны, не скрыты)
- [ ] Given: пользователь переключается с июня (5/3) на июль (0/2) — When: месяц переключён — Then: счётчики обновляются атомарно до новых значений без промежуточного мигания
- [ ] Given: пользователь ставит задачу выполненной в ячейке — When: toggle — Then: «Выполнено» +1, «В работе» −1 немедленно
- [ ] Given: пользователь снимает выполнение — When: toggle — Then: «В работе» +1, «Выполнено» −1 немедленно
- [ ] Given: новая задача добавлена в ячейку текущего месяца — When: задача создана — Then: «В работе» +1 немедленно
- [ ] Given: задача удалена (была completed = false) — When: удаление — Then: «В работе» −1 немедленно
- [ ] Given: задача удалена (была completed = true) — When: удаление — Then: «Выполнено» −1 немедленно
- [ ] Given: задача относится к дню-хвосту (например, 30 июня видна в сетке июля) — When: пользователь смотрит на июль — Then: эта задача не учитывается в счётчиках июля
- [ ] Given: декабрь → переключение на январь — When: месяц переключён — Then: счётчики считают задачи января следующего года, заголовок показывает «Январь 2027»

## Вне рамок (Out of scope)

- Временны́е слоты внутри дня (это todo-лист, не event-менеджер)
- Drag & drop задачи между ячейками (перенос задачи на другую дату мышью)
- Редактирование текста уже созданной задачи
- Режимы отображения: неделя, день, список (только месяц)
- Переход в детальный вид дня по клику (кроме мобильной адаптации, где это необходимо)
- Повторяющиеся задачи (рекуррентность)
- Уведомления и напоминания
- Фильтрация и поиск по задачам
- Цветовая маркировка задач / категории
- Синхронизация между устройствами и вкладками
- Аутентификация
- Экспорт (iCal, CSV и т.д.)
- Пакетные операции («удалить все выполненные»)
- Задачи без даты («когда-нибудь» / Backlog)

## Открытые вопросы / риски

1. **Лимит задач в ячейке**: сколько задач показывать до «+K ещё»? Предложение — 3. Зависит от итоговой высоты ячейки; архитектор может скорректировать под вёрстку.

2. **«+K ещё» — кликабельна ли строка?** Если да — что показывается при клике: раскрытие ячейки вниз (сдвигает остальные строки) или всплывающий попап? Первый вариант сложнее в реализации, второй — привычнее по аналогии с Google Calendar.

3. **Видимость кнопки удаления в ячейке**: всегда видна «×» или появляется только при hover? На мобильных hover нет — кнопка должна быть всегда. На десктопе постоянная кнопка визуально загромождает ячейку.

4. **Поведение при добавлении задачи в «хвост» (день вне текущего месяца)**: переключить месяц автоматически или оставить пользователя на текущем? Автопереключение может дезориентировать.

5. **Мобильная адаптация до 480px — детальный экран дня**: выдвижная панель (sheet) снизу или отдельный экран (роутинг)? Без роутера (текущий Vanilla TS) sheet проще.

6. **Миграция localStorage**: присвоить старым задачам сегодняшнюю дату — принятое решение? Альтернатива: вывести их в отдельный «Без даты» раздел (но он вне рамок этой итерации).

7. **Риск схемы данных**: изменение структуры `Todo` (добавление `dueDate`) требует версионирования схемы localStorage. Если не добавить поле `version` сейчас, следующее изменение структуры снова потребует миграции без надёжного механизма обнаружения.
