# Todo List — Базовые операции (MVP)

## Проблема и цель

Пользователю нужен простой инструмент для отслеживания личных задач в браузере.
Цель MVP — дать возможность добавлять задачи, фиксировать их выполнение и удалять ненужные.
Никаких зависимостей от внешних сервисов, минимальный порог входа.

---

## Пользовательские истории

- Как пользователь, я хочу добавить новую задачу, чтобы не забыть, что нужно сделать.
- Как пользователь, я хочу отметить задачу выполненной, чтобы видеть прогресс и отделять сделанное от несделанного.
- Как пользователь, я хочу снять отметку выполнения с задачи, чтобы исправить ошибочную отметку.
- Как пользователь, я хочу удалить задачу, чтобы убрать из списка то, что больше не актуально.
- Как пользователь, я хочу видеть весь список задач при открытии страницы, чтобы сразу понимать, что запланировано.

---

## Функциональные требования

### FR-1. Добавление задачи
- Интерфейс содержит текстовое поле для ввода названия задачи и элемент подтверждения (кнопка или клавиша Enter).
- Задача создаётся со статусом "не выполнена".
- После добавления поле ввода очищается.
- Нельзя добавить задачу с пустым или состоящим только из пробелов названием.
- Максимальная длина названия задачи — 255 символов.

### FR-2. Отметка задачи выполненной / невыполненной
- Каждая задача имеет переключатель (toggle) статуса выполнения.
- Повторное нажение снимает отметку (toggle, а не односторонняя операция).
- Выполненные задачи визуально отличаются от невыполненных (например, зачёркнутый текст).

### FR-3. Удаление задачи
- У каждой задачи есть элемент управления для удаления.
- После удаления задача мгновенно исчезает из списка.
- Удаление необратимо (в рамках MVP — без корзины и отмены).

### FR-4. Отображение списка
- Все задачи отображаются в порядке добавления (новые снизу).
- Если задач нет — отображается пустое состояние с подсказкой.
- Счётчик оставшихся (невыполненных) задач виден на странице.

### FR-5. Персистентность данных
- Задачи сохраняются между перезагрузками страницы (localStorage или серверное хранилище — на усмотрение архитектора).

## Нефункциональные требования

| Категория       | Требование |
|-----------------|------------|
| Производительность | Операции добавления, отметки и удаления отображаются мгновенно (< 100 мс отклик UI) |
| Доступность     | Управление возможно с клавиатуры (Tab-навигация, Enter для подтверждения, Delete/Backspace не обязателен) |
| Совместимость   | Работает в актуальных версиях Chrome, Firefox, Safari (последние 2 мажорные версии) |
| Адаптивность    | Корректно отображается на экранах от 320px (мобильный) до 1920px (десктоп) |
| Лимиты          | Список поддерживает до 500 задач без деградации UI |
| Надёжность      | Потеря данных при перезагрузке страницы недопустима (требование к персистентности) |

## Критерии приёмки

### Добавление задачи
- [ ] Given: пользователь вводит непустой текст и нажимает Enter / кнопку — When: задача появляется в списке последней — Then: поле ввода очищено, задача имеет статус "не выполнена"
- [ ] Given: пользователь вводит только пробелы — When: нажимает подтверждение — Then: задача не создаётся, сообщение об ошибке или поле подсвечивается
- [ ] Given: пользователь вводит текст длиннее 255 символов — When: пытается добавить — Then: система не позволяет добавить и явно об этом сообщает

### Отметка выполнения
- [ ] Given: задача не выполнена — When: пользователь нажимает переключатель — Then: задача визуально помечена как выполненная
- [ ] Given: задача выполнена — When: пользователь нажимает переключатель — Then: задача возвращается в статус "не выполнена"
- [ ] Счётчик невыполненных задач обновляется корректно при каждом переключении

### Удаление задачи
- [ ] Given: пользователь нажимает кнопку удаления у задачи — When: действие подтверждено — Then: задача исчезает из списка, остальные задачи не изменяются
- [ ] Given: удалена последняя задача — When: список пуст — Then: отображается пустое состояние

### Персистентность
- [ ] Given: пользователь добавил задачи — When: перезагружает страницу — Then: все задачи (со своими статусами) присутствуют в списке

### Пустое состояние
- [ ] Given: нет ни одной задачи — When: пользователь открывает страницу — Then: отображается подсказка (например, "Добавьте первую задачу")

## Вне рамок (Out of scope)

- Аутентификация и учётные записи пользователей
- Синхронизация между устройствами / несколькими вкладками
- Редактирование текста уже созданной задачи
- Категории, теги, приоритеты, дедлайны
- Сортировка и фильтрация (кроме базового порядка добавления)
- Drag-and-drop для изменения порядка
- Отмена последнего действия (Undo/Redo)
- Пакетные операции (удалить все выполненные)
- Уведомления / напоминания
- Экспорт / импорт данных
- Мобильное приложение (только веб)

## Открытые вопросы / риски

1. **Персистентность**: localStorage (только текущий браузер) или серверное API (доступ с любого устройства)? Влияет на сложность реализации на несколько порядков. Рекомендация для MVP — localStorage, сервер — следующая итерация.
2. **Подтверждение удаления**: требуется ли диалог подтверждения ("Вы уверены?") или удаление мгновенное? Без подтверждения — быстрее, но риск случайного удаления выше.
3. **Целевая аудитория**: одиночный пользователь на одном устройстве или несколько? Определяет, нужен ли бэкенд с самого начала.
4. **Локализация**: только русский / только английский / оба? Влияет на разметку и хранение текста.
